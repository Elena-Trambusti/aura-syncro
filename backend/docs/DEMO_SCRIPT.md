# Script demo Aura Syncro — 15 minuti

Usare tenant `aura-demo` (`npm run db:seed-demo`).

## Mostrare (in ordine)

1. **Dashboard** — KPI live, command center
2. **Tavoli** — mappa sala, apri tavolo, ordine cameriere
3. **Collega cliente** — picker CRM sull'ordine (sconto fedeltà)
4. **Cucina** — KDS, stati piatti
5. **Checkout** — carta/contanti, banner POS “in configurazione” (onesto)
6. **Report fiscale** — PDF per regime tenant
7. **Menu QR** — scan demo su `/menu/aura-demo?tavolo=1`

## Non mostrare (o dire “in roadmap / post-setup”)

- Marketing SMS
- AI come differenziatore principale
- SDI live se mock
- Moduli Pro se tenant demo è BASE

## Messaggio POS (onesto)

> «Il POS fisico si collega nella fase di setup, quando ci date marca e modello del registratore. Fino ad allora i pagamenti carta in demo sono simulati; in produzione la ricevuta legale esce dal vostro terminale certificato.»
